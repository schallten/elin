Section 01 // Overview

Handbook

How to use the ELIN language thing.

ELIN is a little stack-oriented language made for running things on very small, low-power hardware. It's not a fully serious language, mostly just an experiment. It compiles code into small bytecode so it can fit on stuff with only 32KB of RAM.

Note: ELIN is mainly tested and built for System Operating. It might not work elsewhere.

If you're new to this, ELIN is a simple way to see how computers work without any of the fancy modern stuff. It's just raw data flow.

Section 02 // Core Concepts

How ELIN Thinks: The Stack

Most modern languages (like Python) let you name every temporary value. ELIN is different. It uses a Stack—a vertical pile of data where you can only interact with the very top item.

The Plate Analogy

Imagine a stack of dinner plates. To add two numbers (5 and 10):

  1. You PUSH 10 onto the stack.
  2. You PUSH 5 onto the stack (it sits on top of 10).
  3. You call ADD. The computer "pops" the top two plates, adds them, and places a new plate with "15" back on the stack.

This "Push/Pop" model is why ELIN is so fast. The CPU doesn't have to search for names in memory; it always knows that the data it needs is at the very top of the stack.

Section 03 // Environment Setup

Getting Started with ELIN

To start programming in ELIN, you'll need the compiler and optionally the visualizer.

Requirements

  • Python 3.6+ for the compiler
  • A web browser for the visualizer

Installation

Clone the repository and navigate to the compiler directory:

git clone https://github.com/schallten/elin.git
cd elin/compiler
python main.py your_program.elin

This will generate a .outz file that can be run on the VM or visualized.

Section 04 // Implementation

Basic Syntax

ELIN combines the readability of modern code with the strictness of professional engineering. Every statement is evaluated line-by-line.

# This is a comment
let int x = 10;
let int y = 20;

let int result = x + y;
print result;

# Update existing variable
x = 30;

Recent updates to ELIN (v0.2.0) allow for the optional use of ; (semicolons) to clearly terminate your thoughts, making the code more robust against whitespace errors.

Section 05 // Memory

Memory & Types

In low-power systems, memory is a precious resource. ELIN uses fixed types to ensure the computer knows exactly how many "slots" to reserve for your data.

Type Storage Use Case
int 64-bit Integer Whole numbers, loop counters, math.
str Pool Pointer Text display and logging prefixes.
arr Fixed Buffer Lists of numbers (e.g. sensor history).
Section 06 // Mathematics

Mathematical Operations

ELIN supports standard arithmetic operations on integers: addition (+), subtraction (-), multiplication (*), and division (/). Operations follow stack-based evaluation.

Basic Math

let int a = 10;
let int b = 5;
let int sum = a + b;  # 15
let int diff = a - b; # 5
let int prod = a * b; # 50
let int quot = a / b; # 2

Complex Expressions

You can nest operations and function calls:

let int result = (a + b) * (a - b);  # 75

All math is performed on 64-bit integers. Division truncates towards zero.

Modulo

The % operator gives you the remainder after division:

let int a = 17;
let int b = 5;
let int r = a % b;  # 2  (17 / 5 = 3 remainder 2)

Absolute Value

abs(x) strips the sign off a number, always returning a non-negative result:

let int x = -42;
let int y = abs(x);  # 42
let int z = abs(5);   # 5
Section 07 // Stack Manipulation

Stack Operations

ELIN gives you direct control over the stack with a handful of low-level instructions. These let you rearrange values without going through variables.

Duplicate & Drop

dup copies the top value and pushes the copy. drop removes the top value entirely:

# Stack: [10]
dup;       # Stack: [10, 10]  — top value doubled
drop;      # Stack: [10]      — back to one

Swap

swap exchanges the top two values on the stack:

# Stack: [10, 20]
swap;      # Stack: [20, 10]  — positions flipped

Negation & Logical Not

neg flips the sign of the top value. not turns zero into one, and anything else into zero:

# Stack: [42]
neg;       # Stack: [-42]
not;       # Stack: [0]       — -42 is non-zero, so NOT gives 0

# Stack: [0]
not;       # Stack: [1]       — zero becomes 1

Increment & Decrement

inc and dec add or subtract 1 from a variable in a single instruction:

let int count = 5;
inc count;  # count is now 6
dec count;  # count is now 5
Section 08 // Conditionals

Decision Making

ELIN supports basic branching via if blocks, allowing your script to make choices based on program state.

let int value = 5;
if value > 0;
    print "Positive";
else;
    print "Negative or Zero";
end;

Under the hood, these compile down to JZ (Jump if Zero) instructions, gracefully skipping sections of bytecode when conditions aren't met.

Section 09 // Loops (While)

Repetition

Use while loops to repeat actions until a condition fails. Combine with conditionals to build complex state machines.

let int limit = 3;
let int i = 0;
while i < limit;
    print i;
    i = i + 1;
wend;

Because ELIN operates close to the hardware, infinite loops will stall the host CPU. Always make sure your loop condition will eventually be false.

Section 10 // Functions & Scoping

Functions & Scoping

Functions allow you to compartmentalize logic. In ELIN, every function has its own Private Memory (a Local Frame). Variables defined inside func stay inside func. 

func int double int value;
    let int result = value * 2;
    return result;
end;

print double 5; # Outputs 10

Attempting to access a variable defined inside a function from the "Main" script will result in a Compilation Error. This prevents "Side Effects" that can crash small hardware.

Section 11 // Input & Output

Reading and Writing

Numeric Input

input() reads a number from the keyboard while the program is running and pushes it onto the stack. Use it to make interactive programs:

print "Enter a number:";
let int x = input();
print "You typed:";
print x;

input() blocks until you type a number and press Enter. The VM expects a valid integer — any non-numeric input will cause a runtime error.

String Input

read() reads an entire line of text from stdin into the string pool and pushes its handle onto the stack. Unlike input(), it accepts any text, not just numbers:

print "What is your name?";
let str name = read();
print strcat("Hello, ", name);

Writing Without Newlines

The print statement always appends a newline. Use write to output a string handle without a newline — useful for prompts:

write "Enter name: ";
flush;
let str name = read();
print strcat("Hello, ", name);

Flushing Output

flush forces any buffered output to be written immediately. Combine with write for prompt-style interactions where the output must appear before blocking on input.

Section 12 // String Operations

Manipulating Strings at Runtime

ELIN provides four built-in operations for text manipulation. All operate on string pool handles (the values stored in str variables).

String Length

strlen(s) pushes the length of the string referenced by handle s:

let str msg = "hello";
let int n = strlen(msg);
print n;  " 5

String Concatenation

strcat(a, b) concatenates two strings and pushes a new handle to the result. The original strings are unchanged:

let str a = "hello ";
let str b = "world";
let str c = strcat(a, b);
print c;  " hello world

Substring

substr(s, offset, length) extracts a portion of a string and pushes a new handle. Offsets and lengths are clamped to the string bounds — out-of-range values won't crash:

let str s = "hello world";
let str a = substr(s, 0, 5);   " "hello"
let str b = substr(s, 6, 5);   " "world"
let str c = substr(s, 0, 100); " "hello world" (clamped)

String Comparison

strcmp(a, b) compares two strings lexicographically and pushes -1 if a < b, 0 if equal, or 1 if a > b:

print strcmp("abc", "def");  " -1
print strcmp("xyz", "xyz");  "  0
print strcmp("zzz", "aaa");  "  1

All string operations work with string pool handles, not raw pointers. The string pool grows at runtime to accommodate new strings created by read(), strcat(), and substr().

Section 13 // Debug Mode

Tracing Execution

The VM has a debug mode that prints every instruction as it runs — opcode, stack contents, and call depth. Launch it with the --debug flag:

./elin_vm --debug program.outz

You can also toggle tracing from inside an ELIN program with trace:

trace;    # Turn debug output ON
let int x = 42;
print x;
trace;    # Turn debug output OFF

This is invaluable when you're trying to understand why the VM produced a wrong result — it's like lifting the hood and watching every gear turn.
